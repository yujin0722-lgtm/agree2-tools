# AGREE II 診療ガイドライン評価ツール

診療ガイドラインをAGREE II(Appraisal of Guidelines for Research and Evaluation II)の日本語訳(Minds, 2022年9月改訂)に基づいて評価するための、Webベースの評価・集計ツールです。Googleフォームを使わず、Google Apps Script(GAS)とGoogle スプレッドシートをバックエンドに、GitHub Pagesでホストした静的HTMLページから利用します。

勉強会などのグループワークで、複数の班(2〜4班)がそれぞれAGREE IIの23項目+全体評価に回答し、その結果をダッシュボードでレーダーチャートやコメント一覧として確認できます。

## 公開ページ

GitHub Pagesを有効にすると、以下のURLで利用できます(`(ユーザー名)`と`(リポジトリ名)`は実際の値に置き換えてください)。

| ページ | URL | 共有範囲 |
|---|---|---|
| ハブページ | `https://(ユーザー名).github.io/(リポジトリ名)/` | 誰でも(参加者・関係者) |
| 評価フォーム | `https://(ユーザー名).github.io/(リポジトリ名)/hyoka.html` | 参加者 |
| ダッシュボード | `https://(ユーザー名).github.io/(リポジトリ名)/dashboard.html` | 参加者・関係者 |
| 管理画面 | `https://(ユーザー名).github.io/(リポジトリ名)/kanri.html` | **主催者のみ(共有しない、ただしindex.html最下部に控えめなリンクあり)** |

## ファイル構成

```
├── index.html          # ハブページ(評価フォーム・ダッシュボードへの入口、使い方の説明、管理画面への控えめなリンク)
├── hyoka.html           # 評価フォーム(参加者が班ごとに回答)
├── dashboard.html        # 結果ダッシュボード(レーダーチャート、コメント一覧、PDF出力、過去の勉強会結果の閲覧)
├── kanri.html            # 管理画面(設定・回答状況・履歴の管理など)
├── Code.gs               # バックエンド(Google Apps Scriptにコピーして使用)
└── セットアップ手順.md     # 初回セットアップの手順書(非エンジニア向け)
```

`Code.gs`はこのリポジトリには実行環境がなく、Google スプレッドシートに紐づくApps Scriptプロジェクトに貼り付けて使います。詳細は`セットアップ手順.md`を参照してください。

## 初めて使う方へ

セットアップは最初の1回だけ必要です。手順は `セットアップ手順.md` にまとめてあります。プログラミングの知識がなくても進められる内容です。

セットアップが終わったら、`kanri.html` の「はじめての方へ」パネルから架空のガイドラインとダミーの班データを生成し、評価フォーム・ダッシュボードの動作を試すことができます。確認が終わったら、同じ画面の「新しい勉強会を開始する」でサンプルデータを整理してから、本番のガイドライン名・班数を設定してください。

以降の日常の運用(設定変更、回答状況の確認、勉強会終了後の処理)はすべて管理画面(`kanri.html`)から行えます。スプレッドシートを直接開く必要はありません。

## 使い方の概要

- **参加者**: `hyoka.html` を開き、班を選んで23項目+全体評価に回答します。各項目には選択を取り消す「クリア」ボタンがあります。「一時保存」で途中保存、「提出(確定)」で確定します。確定後はフォームがロックされ編集できなくなり、修正するには「確定を取り消す」を押す必要があります。
- **関係者・参加者**: `dashboard.html` で、班ごとの領域別得点(レーダーチャート)、コメント、未回答項目の有無を確認できます。PDFとして保存も可能です。画面上部の「表示するデータ」から、過去にアーカイブした勉強会の結果を選んで見返すこともできます。
- **主催者**: `kanri.html` で、ガイドライン名・班数の設定、班数変更時のデータ整理、勉強会終了後の処理(履歴に保存してリフレッシュ/履歴に残さず削除)、過去の履歴の個別削除などを行います。

## kanri.html(管理画面)でできること

1. **サンプルデータを試す**: 架空のガイドラインとダミーの班データを生成し、初めての操作でも動作を確認できます。
2. **勉強会の設定**: ガイドライン名・班数(2〜4)を保存します。班数を減らす場合は、対象外になる班のデータ削除を確認したうえで実行します。
3. **現在の回答状況**: 班ごとのステータス(下書き/確定)・最終更新日時を確認できます。班数の範囲外にデータが残っている場合は警告とクリーンアップボタンが表示されます。
4. **新しい勉強会を開始する**: 現在の回答を「評価履歴」に保存したうえで回答エリアをクリアし、ガイドライン名をリセットします(班数は維持されます)。
5. **回答データを削除する(履歴に残さない)**: 設定ミスやサンプルデータを、記録を残さずに削除します。確認のため所定の文字列の入力が必要です。
6. **過去の勉強会の履歴を管理する**: アーカイブ済みの勉強会が一覧表示され、個別に削除できます。
7. **評価履歴をすべて削除する(特殊操作)**: 全履歴を完全に削除する、通常は使わない特殊操作です。確認のため所定の文字列の入力が必要です。

## 技術構成

- **フロントエンド**: 素のHTML/CSS/JavaScript(ビルド不要、GitHub Pagesでそのまま配信)
- **バックエンド**: Google Apps Script によるWebアプリ(`doGet`/`doPost`)
- **データストア**: Google スプレッドシート(設定・回答・評価履歴の3シート)
- **グラフ描画**: Chart.js(レーダーチャート)
- **PDF出力**: html2canvas + jsPDF

## 注意事項

- `kanri.html` のURLは基本的に参加者に共有しないでください。アクセス制限(パスワードなど)は設けていないため、URLを知っている人は誰でも設定変更ができます(index.html最下部のリンクも、参加者が誤って操作しない程度の配慮であり、アクセス制限ではありません)。
- Code.gsを更新した場合は、Apps Scriptで「新しいデプロイ」を作成しないと変更が反映されません。HTMLファイル(index.html/hyoka.html/dashboard.html/kanri.html)はGitHubにアップロードするだけで反映されます。
- 過去の勉強会の詳細な回答は、スプレッドシートの「評価履歴」シートに保存されます。個別の削除・全削除は`kanri.html`から行えます。
