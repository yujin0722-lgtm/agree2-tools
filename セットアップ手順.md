# AGREE II 評価ツール セットアップ手順

この手順は一度だけ行えば完了します。以降、勉強会のたびに繰り返す必要はありません。
お渡ししたファイルは6つです。

- `Code.gs`(スプレッドシートに貼り付けるコード)
- `index.html`(トップページ。参加者や関係者への案内はここから)
- `hyoka.html`(参加者が使う評価フォーム)
- `dashboard.html`(結果を確認するダッシュボード。関係者に共有してよいもの)
- `kanri.html`(主催者専用の管理画面。**このURLは基本的に参加者には共有しないでください**)
- `README.md`(このツール全体の説明。GitHubリポジトリのトップに置くと概要が表示されます)

このセットアップを終えれば、以降の日常の運用(ガイドライン名や班数の設定、勉強会終了後のリフレッシュなど)はすべて `kanri.html` の管理画面から行えます。スプレッドシートを直接開く必要があるのは、今回のセットアップ作業と、万一のトラブル対応のときだけです。

順番に進めてください。途中で画面が想定と違う場合は、その時点で教えてください。

---

## ステップ1: スプレッドシートを準備する

1. Google スプレッドシートで新しいスプレッドシートを作成します。名前は自由です(例:「AGREE II 評価ツール」)。
2. 上部メニューの「拡張機能」→「Apps Script」をクリックします。新しいタブでコード編集画面が開きます。
3. 画面に最初から書かれているコード(`function myFunction() {}` など)を全て選択(Ctrl+A、Macは Cmd+A)して削除します。
4. お渡しした `Code.gs` の中身を全部コピーして、空になった編集画面に貼り付けます。
5. 画面左上のフロッピーディスクのアイコン(保存)をクリックします。

---

## ステップ2: シートを自動作成する

1. Apps Script の編集画面の上部にある関数選択のプルダウン(「デプロイ」の左あたり)で `setupSheets` を選びます。
2. その左にある「実行」ボタン(▷のアイコン)をクリックします。
3. 初回は「承認が必要です」という画面が出ます。「権限を確認」→ご自身のGoogleアカウントを選択→「詳細」というリンクをクリック→一番下の「(プロジェクト名)に移動(安全ではないページ)」をクリック→「許可」をクリックします。これはご自身が作ったコードなので問題ありません。
4. 元のスプレッドシートのタブに戻り、画面を再読み込み(更新)します。上部に「AGREE II 評価ツール」という新しいメニューが表示されます。
5. 念のため、そのメニューから「初期セットアップ(シートを作成)」をクリックします。「設定」「回答」「評価履歴」という3つのシートが追加されます。

---

## ステップ3: 管理画面用の「合言葉」を決める

管理画面(`kanri.html`)の操作(設定変更・データ削除・履歴管理など)は、決められた「合言葉」が一致しないと実行できない仕組みになっています。これは、URLを知っているだけの第三者が管理操作を行えてしまうのを防ぐためです。

1. 20文字程度の、推測されにくいランダムな英数字を決めます(パスワード管理ソフトの生成機能や、キーボードを適当に打った文字列で構いません。例: `x7Qp2mZk9Lw4Rt8Vn3Bc`)。
2. Apps Script の編集画面に戻り、`Code.gs` の上のほうにある次の行を探します。

   ```
   var ADMIN_KEY = 'REPLACE_THIS_WITH_YOUR_OWN_SECRET';
   ```

3. `REPLACE_THIS_WITH_YOUR_OWN_SECRET` の部分を、決めた合言葉に置き換えます(前後の引用符 `'` は残してください)。
4. 保存します。この合言葉は、後述の `kanri.html` にも同じものを設定します。忘れないようメモ帳などに控えておいてください。

---

## ステップ4: Webアプリとして公開する

1. Apps Script の編集画面に戻り、右上の青い「デプロイ」ボタン→「新しいデプロイ」をクリックします。
2. 「種類の選択」の横にある歯車アイコンをクリックし、「ウェブアプリ」を選びます。
3. 「説明」欄は空欄のままで構いません。
4. 「次のユーザーとして実行」は「自分」のままにします。
5. 「アクセスできるユーザー」は「全員」を選びます(これにより、参加者がログインなしで使えるようになります)。
6. 「デプロイ」ボタンをクリックします。
7. 再び承認画面が出ることがあります。ステップ2と同じように「アクセスを承認」を進めてください。
8. 「ウェブアプリ」という項目の下に表示される **URL(`https://script.google.com/macros/s/....../exec` という形式)をコピー**して、メモ帳などに控えておいてください。これが今後何度も使う「API URL」です。

---

## ステップ5: GitHub Pages を用意する

GitHubアカウントをまだお持ちでない場合は、[github.com](https://github.com) で無料登録してください(メールアドレスがあれば数分で完了します)。

1. GitHubにログインし、右上の「+」→「New repository」をクリックします。
2. リポジトリ名を入力します(例: `agree2-tool`)。「Public」を選んだままにします。
3. 「Create repository」をクリックします。
4. 作成されたリポジトリの画面で「Add file」→「Upload files」をクリックします。
5. お渡しした `index.html`・`hyoka.html`・`dashboard.html`・`kanri.html`・`README.md` の5つのファイルをドラッグ&ドロップ、または選択してアップロードします。
6. 下部の「Commit changes」をクリックして保存します。

### API URLと合言葉を設定する

1. リポジトリ画面で `hyoka.html` をクリックして開き、右上の鉛筆アイコン(Edit)をクリックします。
2. コードの中から `YOUR_WEB_APP_URL_HERE` という文字列を探します(上のほうにあります)。
3. その部分を、ステップ4でコピーしたAPI URLに置き換えます(前後の引用符 `"` は残してください)。
4. 右上の「Commit changes」をクリックして保存します。
5. 同じ作業を `dashboard.html` でも行います(`YOUR_WEB_APP_URL_HERE` を同じAPI URLに置き換える)。
6. `kanri.html` では、2箇所書き換えが必要です。`YOUR_WEB_APP_URL_HERE` を同じAPI URLに置き換えることに加えて、`YOUR_ADMIN_KEY_HERE` という文字列を、ステップ3で決めた合言葉に置き換えます(Code.gsに書いた合言葉と完全に一致させてください)。

`index.html` はAPI URLを使わないため、書き換え不要です。

### GitHub Pages を有効にする

1. リポジトリ画面上部の「Settings」タブをクリックします。
2. 左側のメニューから「Pages」を選びます。
3. 「Branch」の項目で、「main」と「/ (root)」を選び、「Save」をクリックします。
4. 数分待つと、ページ上部に公開URLが表示されます(`https://(あなたのユーザー名).github.io/agree2-tool/` のような形式)。

これで、以下のURLが使えるようになります。

- トップページ(誰でも): `https://(ユーザー名).github.io/agree2-tool/`
- 評価フォーム(参加者に共有): `https://(ユーザー名).github.io/agree2-tool/hyoka.html`
- ダッシュボード(関係者に共有可): `https://(ユーザー名).github.io/agree2-tool/dashboard.html`
- 管理画面(主催者のみ、基本的に共有しない): `https://(ユーザー名).github.io/agree2-tool/kanri.html`

すべて忘れないようメモ帳などに控えておいてください。

---

## ステップ6: サンプルデータで動作を試す

1. `kanri.html` のURLを開きます。
2. 一番上の「はじめての方へ」パネルにある「サンプルデータを生成する」を押します。合言葉が正しく設定されていれば、架空のガイドラインとダミーの班データが作成されます。
3. 評価フォーム・ダッシュボードのURLを開き、それぞれ正しく表示されるか確認します。
4. 確認が終わったら、`kanri.html` の「3. 新しい勉強会を開始する」を押して、サンプルデータを整理します(ガイドライン名は自動的に空欄に戻ります)。

もしここでエラーが出る場合は、`hyoka.html`・`dashboard.html`・`kanri.html` の中のAPI URL、および `kanri.html` の合言葉(ADMIN_KEY)が、Code.gs側と正しく一致しているかご確認ください。

---

## ステップ7: 本番の勉強会を設定する

1. `kanri.html` の「1. 勉強会の設定」で、今回評価するガイドラインの名前を入力します。
2. 「班数」で2〜4のいずれかを選びます。
3. 「この内容で保存する」をクリックします。「保存しました」と表示されれば成功です。
4. 評価フォームを開き、ガイドライン名と班数が正しく反映されているか確認します。

---

## 日常の使い方

- **参加者**: トップページ、または評価フォームのURLにアクセスし、班を選んで回答するだけです。デプロイやApps Scriptの操作、スプレッドシートを開く操作は一切不要です。
- **主催者**: 次のガイドラインを評価する際は、管理画面(`kanri.html`)でガイドライン名・班数を設定するだけです。勉強会が完全に終わったら、管理画面の「新しい勉強会を開始する」ボタンを押してください。今回のデータは自動的に「評価履歴」シートに保存され、回答エリアとガイドライン名がリセットされます。過去の履歴は、ダッシュボードの「表示するデータ」から見返せるほか、管理画面から個別に削除することもできます。スプレッドシートを直接開く必要はありません。
- **スプレッドシートを開く場面**: 管理画面が万一使えないときの予備手段(スプレッドシートのメニュー「AGREE II 評価ツール」からも一部の操作ができます)くらいです。

## Code.gsを更新したとき

今後、機能追加などで `Code.gs` の内容を更新した場合は、貼り付けて保存するだけでなく、Apps Scriptの「デプロイを管理」→編集(鉛筆アイコン)→「バージョン」で「新バージョン」を選び、再度「デプロイ」を押す必要があります。これを忘れると、変更が公開中のWebアプリに反映されません。`index.html`・`hyoka.html`・`dashboard.html`・`kanri.html` などのHTMLファイルは、GitHubにアップロードするだけで反映されます。
